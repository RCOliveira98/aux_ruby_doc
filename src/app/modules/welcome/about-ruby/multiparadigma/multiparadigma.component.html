Multiparadigma...

Paradigma nada mais é do que um estilo de programação. Uma linguagem multiparadigma caracteriza-se por agrupar mais de um paradigma de programação para assim melhor atender as nossas necessidades, dessa forma, podemos trabalhar com vário estilos de programação usando uma única linguagem.

Ruby abrange de forma equilibrada os paradigmas de programação imperativa, funcional e orientado a objetos.

Imperativa: é um paradigma aliado a arquitetura de computadores, arquitetura que foi criada por von Neumann. Diversas linguagens de programação foram projetadas seguindo esse paradigma. Algumas catacterísticas das linguagens imperativas são o uso de variáveis, atribuição de valores e fluxo de execução sequencial de comandos.

Funcional: é um paradigma de programação baseado em chamadas de funções e que evita estados ou dados mutáveis. Prega pelo uso de constantes e não de variáveis, é declarativa e não imperativa. Atualmente, é um dos paradigmas de programação mais populares.

Orientado a objetos: é um paradigma de programação que visa representar o mundo natural dentro do mundo computacional. Algumas de suas características são o uso de classes, herança, polimorfismo, interfaces etc. Ruby é uma linguagem puramente orientada a objetos, tratando tudo como um objeto.
